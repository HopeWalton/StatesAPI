Welcome to the U.S. States REST API

This API provides information about all U.S. states, including fun facts, and allows CRUD operations via URL parameters and JSON requests.

GET Requests

  • /states – Returns all state data
  • /states?contig=true – Returns only contiguous states (excludes AK and HI)
  • /states?contig=false – Returns only non-contiguous states (AK and HI)
  • /states/:state – Returns all data for a specific state by its 2-letter code (e.g., /states/TX)
  • /states/:state/funfact – Returns a random fun fact for that state
  • /states/:state/capital – Returns the state’s capital
  • /states/:state/nickname – Returns the state’s nickname
  • /states/:state/population – Returns the population
  • /states/:state/admission – Returns the date the state was admitted to the Union

POST Request

/states/:state/funfact – Add one or more fun facts to a state

Body (JSON):

{
  "funfacts": [
    "Fact one.",
    "Fact two."
  ]
}

PATCH Request

/states/:state/funfact – Update a specific fun fact

Body (JSON):

{
  "index": 1,
  "funfact": "Updated fun fact."
}

DELETE Request

/states/:state/funfact – Delete a specific fun fact by index

Body (JSON):

{
  "index": 2
}

Notes

  • State codes must be entered as 2-letter abbreviations (e.g., CA, NY, TX).
  • Full state names will result in a 404 error.
  • Use tools like Postman or browser URL bar to test GET requests.
  • Use Postman or similar tools to send POST, PATCH, and DELETE requests with JSON bodies.
